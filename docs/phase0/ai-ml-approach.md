# AI/ML アプローチ（Phase 0）

> Issue: #3
> ステータス: 確定

## 概要

コーデ提案パイプラインは2つの技術を組み合わせる。

1. **Claude API (Vision)** — 画像の意味理解・コーデ提案文生成
2. **CLIP + pgvector** — 画像の視覚的類似度検索

---

## パイプライン詳細

### Step 1: クローゼット服の登録時

```
服の画像アップロード
    ↓
[Claude API Vision]
  - カテゴリ自動判定 (tops / bottoms / outerwear / shoes / bag / accessory)
  - 色抽出 (hex array)
  - タグ生成 (例: "カジュアル", "ストライプ", "コットン")
    ↓
[CLIP ViT-B/32]
  - 512次元の画像埋め込みベクトル生成
    ↓
PostgreSQL (pgvector) に保存
```

### Step 2: コーデ提案生成

```
インスピレーション画像入力
    ↓
[Claude API Vision]
  - スタイル分析 (カジュアル / フォーマル / ストリート 等)
  - アイテム分解 (トップス・ボトムス・シューズ・アウター等のカテゴリ別に特徴を記述)
  - 色パレット抽出
    ↓
[CLIP ViT-B/32]
  - インスピレーション画像の埋め込みベクトル生成
    ↓
[pgvector コサイン類似度検索]
  - カテゴリ別にユーザー手持ち服からTop-3を取得
    ↓
[Claude API]
  - 取得した手持ち服の組み合わせの整合性評価
  - コーデの説明文・スタイルノート生成
  - 不足カテゴリの特定
    ↓
コーデ提案 (最大3パターン) を返却
```

### Step 3: ファストファッション商品提案

```
不足カテゴリ + インスピレーション画像の特徴
    ↓
[楽天市場 API] カテゴリ・カラーで商品検索
    ↓
[CLIP] 商品画像のベクトル化（非同期バッチ）
    ↓
[pgvector] インスピレーション画像に近い商品を類似度順でソート
    ↓
Top-3 商品を提案
```

## Claude API プロンプト設計

### 服のタグ付け

```
system: あなたはファッションの専門家です。
        服の画像を分析し、JSON形式で情報を返してください。

user: [服の画像]
      以下のJSON形式で分析結果を返してください：
      {
        "category": "tops|bottoms|outerwear|shoes|bag|accessory|other",
        "colors": ["#hex1", "#hex2"],
        "tags": ["タグ1", "タグ2", ...],
        "brand": "ブランド名（不明の場合は null）"
      }
```

### コーデ提案

```
system: あなたはファッションコーディネーターです。
        初心者でも理解できる、簡潔で前向きなアドバイスをしてください。

user: [インスピレーション画像]
      このコーデのスタイルを分析し、以下のJSON形式で返してください：
      {
        "style": "スタイルの特徴（100字以内）",
        "palette": ["#hex1", "#hex2"],
        "items": {
          "tops": "トップスの特徴",
          "bottoms": "ボトムスの特徴",
          "outerwear": "アウターの特徴（なければ null）",
          "shoes": "シューズの特徴",
          "bag": "バッグの特徴（なければ null）"
        }
      }
```

## CLIP モデル選定

| モデル | 次元数 | 精度 | 推論速度 | 採用 |
|--------|--------|------|----------|------|
| ViT-B/32 | 512 | ★★★☆ | ★★★★★ | ✅ MVP 採用 |
| ViT-B/16 | 512 | ★★★★ | ★★★☆☆ | 将来検討 |
| ViT-L/14 | 768 | ★★★★★ | ★★☆☆☆ | 将来検討 |

MVP は ViT-B/32 で十分な精度と速度のバランスを取る。
