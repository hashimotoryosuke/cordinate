# ファッションコーディネートアプリ MVP ロードマップ

## プロジェクト概要

ファッション初心者が、自分の手持ち服をベースに「おしゃれなコーデ」を簡単・短時間で組めるようにするアプリ。
ファッションショー・雑誌・インフルエンサーのコーデを参考に、似せたスタイルを提案する。

- **プラットフォーム**: Web + スマホアプリ（iOS / Android）
- **ターゲット**: ファッションに自信がない初心者
- **コア価値**: 時間をかけずに、自分の服でおしゃれなコーデが組める

---

## MVP スコープ

MVP では「コーデ提案」の核心機能に絞り、早期ユーザー検証を優先する。

### MVP に含める機能

| # | 機能 | 説明 |
|---|------|------|
| 1 | ユーザー登録・認証 | メール / SNS ログイン |
| 2 | クローゼット登録 | 手持ち服を写真撮影 or アップロードで登録 |
| 3 | インスピレーション入力 | ファッションショー・雑誌・インフルエンサーのコーデ画像 URL or アップロード |
| 4 | AI コーデ提案 | 参考コーデに近い組み合わせを手持ち服から提案 |
| 5 | ファストファッション商品提案 | 手持ちで不足するアイテムをユニクロ・ZARA 等から提案 |
| 6 | コーデ保存・一覧 | 提案されたコーデをお気に入り保存 |

### MVP に含めない機能（将来フェーズ）

- SNS シェア・フォロー機能
- コミュニティ・コーデ投稿
- EC 直接購入連携
- 体型・骨格診断パーソナライズ
- AR 試着

---

> **Note**: 実装は Claude Code を使用するため、コード生成・UI実装・API統合は大幅に短縮。
> ただし、設計の意思決定・AI精度チューニング・外部API調査・App Store審査は短縮しにくい。

## フェーズ別ロードマップ

### Phase 0: 準備・設計（3〜5日）

**目標**: 技術選定・アーキテクチャ確定・開発環境構築

#### タスク

- [ ] 技術スタック確定
- [ ] DB スキーマ設計（ユーザー・服・コーデ）
- [ ] AI/ML アプローチ選定（画像類似検索、コーデ提案）
- [ ] 外部 API 調査（ファストファッション商品データ取得方法）
- [ ] デザインシステム・UI ワイヤーフレーム作成
- [ ] 開発環境 / CI/CD 構築
- [ ] リポジトリ構成決定（モノレポ推奨）

#### 技術スタック候補

| レイヤー | 技術 |
|----------|------|
| フロントエンド Web | Next.js (App Router) + TypeScript |
| モバイル | React Native (Expo) |
| バックエンド | Node.js / Hono or FastAPI (Python) |
| DB | PostgreSQL + pgvector（画像ベクトル検索） |
| 画像ストレージ | AWS S3 or Cloudflare R2 |
| AI 画像解析 | Claude API (claude-opus-4-6) or GPT-4o Vision |
| 画像埋め込みベクトル | CLIP モデル |
| 商品データ | スクレイピング or アフィリエイト API |
| 認証 | Supabase Auth or Auth.js |
| インフラ | Vercel (Web) + ECS or Fly.io (API) |

### Phase 1: コア基盤構築（4〜7日）

**目標**: ユーザー認証・クローゼット登録・基本 API を完成させる

#### バックエンド

- [ ] ユーザー認証 API（登録・ログイン・JWT）
- [ ] クローゼット CRUD API（服の登録・取得・削除）
- [ ] 画像アップロード API（S3/R2 署名付き URL）
- [ ] 画像解析パイプライン
  - 服の種類・色・素材をAI で自動タグ付け
  - CLIP で画像埋め込みベクトル生成・pgvector に保存

#### フロントエンド（Web）

- [ ] 認証画面（ログイン・新規登録）
- [ ] クローゼット登録画面（カメラ撮影 / ファイルアップロード）
- [ ] クローゼット一覧画面（サムネイルグリッド）

#### モバイル（Expo）

- [ ] 認証画面
- [ ] クローゼット登録画面（カメラ権限・ネイティブカメラ連携）

**完了基準**: 服を登録し、一覧で確認できる状態

---

### Phase 2: AI コーデ提案機能（1.5〜2週間）

**目標**: 参考コーデ入力 → 手持ち服からのコーデ提案を実装

#### AI パイプライン

- [ ] 参考コーデ画像の解析
  - アイテム分解（トップス・ボトムス・シューズ・アクセ）
  - 色・スタイル・シルエット抽出
- [ ] 手持ち服とのマッチング
  - pgvector で類似度検索（コサイン類似度）
  - Claude API でコーデの整合性スコアリング
- [ ] コーデ提案生成
  - マッチした手持ち服の組み合わせを複数パターン提案
  - 各提案に「なぜこのコーデが似合うか」の説明文生成

#### バックエンド

- [ ] インスピレーション入力 API（URL / 画像アップロード）
- [ ] コーデ提案 API（非同期処理・ジョブキュー）
- [ ] コーデ保存 API

#### フロントエンド

- [ ] インスピレーション入力画面（URL 貼り付け・画像アップロード）
- [ ] コーデ提案画面（手持ち服の組み合わせを視覚的に表示）
- [ ] コーデ保存・お気に入り一覧

**完了基準**: 参考コーデ画像を入力し、手持ち服でのコーデ提案が3パターン以上表示される

---

### Phase 3: ファストファッション商品提案（3〜5日）

**目標**: 手持ちで不足するアイテムをファストファッションから提案

#### 商品データ取得

- [ ] 商品データソース確定（優先順位）
  1. 楽天市場 API / Yahoo! ショッピング API（アフィリエイト）
  2. ユニクロ・ZARA 等の公開 API or スクレイピング
- [ ] 商品画像の CLIP ベクトル化・インデックス構築

#### 提案ロジック

- [ ] 不足アイテム検出（コーデ提案でマッチしなかったカテゴリを特定）
- [ ] 商品ベクトル検索で類似商品を取得
- [ ] 価格帯・ブランドでフィルタリング

#### フロントエンド

- [ ] コーデ提案画面に「不足アイテムを購入する」セクション追加
- [ ] 商品カード（画像・価格・ブランド・購入リンク）

**完了基準**: コーデに不足するアイテムについて、3件以上の購入候補商品が表示される

## マイルストーン サマリー

> Claude Code による実装を前提とした見積もり。コード生成は大幅短縮されるが、AI精度チューニングや App Store 審査は変わらない。

| マイルストーン | 旧見積 | Claude Code 前提 | 短縮の主な理由 |
|----------------|--------|------------------|----------------|
| Phase 0: 準備・設計 | 2週間 | **3〜5日** | 設計作業自体は短縮しにくいが、環境構築は自動化で短縮 |
| Phase 1: コア基盤 | 3週間 | **4〜7日** | 認証・CRUD・API は Claude Code で数時間〜1日 |
| Phase 2: AI コーデ提案 | 4週間 | **1.5〜2週間** | プロンプト精度チューニングが律速 |
| Phase 3: 商品提案 | 2週間 | **3〜5日** | 外部 API 調査次第だがコード実装は高速 |
| Phase 4: UX 改善 | 2週間 | **5〜7日** | ユーザーテストのフィードバックサイクルが律速 |
| Phase 5: クローズドβ | 1週間 | **1〜2週間** | App Store 審査期間は固定（変わらない） |
| **合計** | **14週間** | **約5〜7週間** | 約 **50〜65% 短縮** |

## リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| AI コーデ提案の精度が低い | 高 | Phase 2 でユーザーテストを早期実施し、プロンプト・モデル調整を繰り返す |
| 商品データの取得困難 | 中 | 楽天 API を第一候補に置き、スクレイピングはフォールバック |
| 画像解析コスト増大 | 中 | バッチ処理・キャッシュで API 呼び出し回数を最小化 |
| App Store 審査リジェクト | 低 | 審査ガイドライン準拠を Phase 4 で確認、早めに申請 |

## 成功指標（MVP 検証基準）

- クローズドβ 100名以上の参加
- コーデ提案に対する満足度: 70% 以上（「参考になった」以上の評価）
- 1ユーザーあたりの平均コーデ提案数: 3回以上/週
- 商品提案からの購入クリック率: 10% 以上
